# Herrschaftsgebiete

This note describes the new data and editor model for political
Herrschaftsgebiete in the existing Avesmaps system.

## Data model

The model separates Wiki reference data, editorial Avesmaps data and
geometries:

- `political_territory_wiki` stores imported Wiki-Aventurica fields
  as unchanged as possible. WikiSync updates this table based on a
  stable key derived from the Wiki link or normalized name.
- `political_territory` is the editorial Avesmaps model. It contains
  display names, color, transparency, hierarchy, timeline, zoom filters,
  Wappen link and editorial notes.
- `political_territory_geometry` contains the actual map geometry as
  GeoJSON `Polygon` or `MultiPolygon`. A Herrschaftsgebiet may have any
  number of geometries; enclaves, exclaves and holes are represented via
  MultiPolygons and polygon rings.

The earlier `political_territory_relation` table has been removed in the
current state. The hierarchy is represented directly via
`political_territory.parent_id`.

The migration lives in `sql/2026-05-15-political-territories.sql`; the same
schema is also contained in `api/schema.future.mysql.sql`. The existing
admin endpoint reports the new tables in the database status.

## WikiSync

The normal WikiSync run imports Herrschaftsgebiete in a dedicated phase
`political_territories`. The server-side reference lives in the table
`political_territory_wiki`; local JSON files are no longer used for this.

On import, only records with `Kontinent = Aventurien` are created as local
Herrschaftsgebiete. Wiki reference data is updated, but editorial fields
and existing geometries are not overwritten.
The imported reference contains 500 Herrschaftsgebiete in 36
`Zugehoerigkeit-Root` areas; the editor uses these root values as the
search and grouping structure in the parent/hierarchy tree.
If a local territory does not yet have an editorial geometry and an old
`region` matches by name, normalized name, geographic/political field or
affiliation path, that region is copied as the starting geometry. Automatically
seeded `legacy_region_seed` geometries may be regenerated by the sync,
editorially saved `editor` geometries may not.

Wappen URLs are imported when they do not look like navigation, map or
placeholder graphics. Blocked patterns include `pfeil`, `arrow`,
`positionskarte`, `karte`, `map`, `icon`, `noimage`, `transparent` and
`region_ohne`.

## API

`api/political-territories.php` provides the new endpoints:

- `GET action=layer`: load visible Herrschaftsgebiete for layer, BF year, zoom and
  optional bounding box.
- `GET action=list`: load selectable Herrschaftsgebiete for the editor.
- `GET action=get`: load a single territory with Wiki reference and geometries.
- `GET action=wiki`: load Wiki reference data for a territory.
- `GET action=wiki_list`: load all Wiki references for the editor picker.
- `GET action=hierarchy`: load the current tree structure.
- `GET action=geometries`: load the geometries of a territory.
- `PATCH action=create_territory|update_territory|delete_territory`: edit the
  editorial model.
- `PATCH action=create_geometry|update_geometry|delete_geometry`: edit
  geometries.
- `PATCH action=save_hierarchy`: save the parent structure.
- `PATCH action=geometry_operation`: persist the result of a union, difference or
  intersection on the server side.

The layer endpoint filters numerically over `valid_from_bf`, `valid_to_bf` as
well as geometry time fields. If parent and child would be visible at the same
time, the parent is deterministically suppressed.

## Frontend and editor

The existing `political` layer loads the new Herrschaftsgebiete model when SQL
is configured. If the API is not reachable, the map falls back to the previous
`map_features` Regionen.

The BF timeline appears only when the political layer is active. The
default value is the current editorial year `1049 BF`. Negative
BF values stand internally for `v. BF`. Textual datings from the Wiki are kept
for display and editorial purposes.

In the editor, Herrschaftsgebiete can be selected, edited via double-click in the
geometry, and processed via a right-click context menu.
The context menu offers:

- Edit boundaries
- Edit properties
- Merge with another
- Cut from another
- Cut a new one from another
- Delete

Active multi-step operations are shown as a chip and can be cancelled with `Esc`.

The properties dialog shows the Wiki reference permanently as a box at the top,
including a clickable Wiki link. Via "Wiki-Referenz aendern", a different
imported Wiki reference can be searched and linked. The parent/hierarchy tree
has a search field, collapsible root/parent nodes and groups the long lists by
the 36 Wiki root areas. Color and transparency are in one row; the Wappen link
shows an immediate preview. Numeric from/to years can be set directly, where
"heute" disables the to field and stores it as an open end.

## Geometry phase

Fully implemented:

- multiple geometries per Herrschaftsgebiet
- `Polygon` and `MultiPolygon` including holes
- snap-to-vertex
- snap-to-edge
- shared-vertex update of adjacent Herrschaftsgebiete when moving shared corner
  points
- union, difference and intersection via `polygon-clipping`
- server-side saving of the computed operation results

Prepared, but not yet implemented as true shared-boundary topology:

- a dedicated relation table for later topology and time relations
- geometry operations that can produce shared boundaries more cleanly
- edge splitting and full shared-edge maintenance without identical corner
  points

Moving a shared corner updates adjacent territories when their corner point
already lies on the same coordinate. Moving entire edges without shared corner
points is the next expansion stage.

## Manual tests

1. Install the schema or run the migration.
2. Open WikiSync in the editor and run "Synchronisieren". The run imports
   locations and then Herrschaftsgebiete.
3. Activate the "Politisch" layer and check that the timeline appears bottom
   center.
4. Change the BF year and check the visible territories.
5. Switch zoom levels and check the parent/child rendering.
6. Double-click a Herrschaftsgebiet, drag corner points and check saving.
7. Right-click a region, choose union/difference/intersection and click a
   second territory.
8. Press `Esc` during an operation and check that the chip disappears.

Local note: The PHP CLI in this working environment has no `mbstring`;
a direct import runtime probe therefore aborts on the existing
`mb_*` helpers. The PHP files were statically checked with `php -l`.
